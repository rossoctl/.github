# The secure platform for AI agents on Kubernetes

<div className="rosso-intro">

rossoctl gives every AI agent a cryptographic identity, scoped access, and isolation — on Kubernetes, on infrastructure you control. It works with agents built in any framework, from LangGraph and CrewAI to any A2A-compatible container, and builds on open standards: A2A between agents, MCP for tools. Open source under Apache 2.0, started at IBM Research with Red Hat.

Deploying an agent is easy. Running an autonomous one — one that acts on its own, calls tools, and rewrites its own workspace — safely, on infrastructure you control, is not. That's the problem we work on, in the open.

## Key capabilities

What rossoctl gives you — and what each lets you do. It works with agents from any framework, runs on Kubernetes, and needs no changes to your agent code.

<div className="row kc-grid">
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">SPIFFE/SPIRE workload identity</div>
      <div className="kc-desc">Every agent gets a rotating cryptographic identity — no shared API keys or static secrets to leak.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">Token exchange — AuthBridge (RFC 8693)</div>
      <div className="kc-desc">Agents act on a user's behalf with short-lived, audience-scoped tokens instead of standing credentials.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">Capability-intersection authorization</div>
      <div className="kc-desc">An agent can only do what the user <em>and</em> the agent are both allowed to — never the union of the two.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">Container + VM sandbox isolation</div>
      <div className="kc-desc">Run self-modifying or untrusted agents without risking the host or other tenants.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">MCP Gateway with auth injection</div>
      <div className="kc-desc">Route every tool call through one authenticated, audited front door — agents never hold tool credentials.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">A2A protocol</div>
      <div className="kc-desc">Agents discover and call each other across frameworks over an open standard.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">Multi-LLM proxy (LiteLLM)</div>
      <div className="kc-desc">Switch or mix 15+ model providers without touching agent code.</div>
    </div>
  </div>
  <div className="col col--6 margin-bottom--md">
    <div className="kc-item">
      <div className="kc-name">Tracing — Phoenix + OpenTelemetry</div>
      <div className="kc-desc">See every LLM call, tool call, and authorization decision correlated in one trace.</div>
    </div>
  </div>
